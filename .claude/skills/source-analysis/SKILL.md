# Source Analysis

- Read session context and uploaded material before inferring source facts.
- Split mixed prompts into distinct scenarios and only attach source facts to the scenario supported by the uploads.
- Capture source systems, formats, ownership, access constraints, and quality risks.
- Update `transformation-understanding.md` with concrete source evidence, assumptions, and missing source-side inputs.
- Label which source claims are verified from uploads versus inferred from context.
- Hand off cleanly when target contract interpretation or mapping work should take over.
