# Validation And Reconciliation

- Design checks that reflect the real transformation risk, not generic QA filler.
- Be explicit about which validations are actually supported by the uploaded evidence and which are placeholders pending more input.
- Cover representative cases, reconciliations, contract checks, and failure paths.
- Update dependency and delivery artifacts when validation work changes the delivery path.
- Be explicit about what evidence would prove the transformation is safe to ship.
