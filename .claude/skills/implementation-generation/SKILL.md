# Implementation Generation

- Confirm the source and target are sufficiently clear before generating implementation output.
- State when implementation output is a partial best-effort deliverable because source or target evidence is incomplete.
- Prefer Python-first deliverables unless the user asks for another target.
- Produce thin, testable implementation slices before broad build plans.
- Keep `delivery-implementation-plan.md` aligned with any generated implementation approach.
