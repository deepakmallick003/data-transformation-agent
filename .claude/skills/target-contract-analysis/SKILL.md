# Target Contract Analysis

- Identify target systems, downstream consumers, delivery modes, and validation expectations.
- If a target schema, DDL, or target contract is missing, say so explicitly instead of inferring load-readiness.
- Separate confirmed target contract facts from assumed acceptance criteria.
- Keep `transformation-understanding.md` current with target-facing constraints and success conditions.
- Flag missing target clarity when it would materially affect implementation or validation.
